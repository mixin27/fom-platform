This section serves two purposes:

- document the current backend API surface
- frame the future API needed for third-party integrations

## Current state

Today, the platform already has a documented backend surface with:

- authenticated operational routes
- shop and platform workspaces
- exports, notifications, realtime tickets, and push-device registration
- public plan catalog endpoints
- documented OpenAPI and Scalar reference URLs

## Future plan

Third-party integration is still a forward-looking capability, but the groundwork is visible.

- response envelopes are already standardized
- resource boundaries are documented
- feature gating and access control are explicit
- API-related docs live beside the broader product and platform reference
